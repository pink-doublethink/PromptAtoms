## Ошибка разметки кода в Python
-----
Традиционное размещение строк документации (docstrings) после объявления функции в Python ухудшает работу GPT-моделей при больших контекстах (более 20 тысяч токенов). 

Из-за особенностей маскированного внимания нейросеть не может "заглядывать" вперед, поэтому семантика описания не связывается с "якорным токеном" (названием) функции. 

Чтобы искусственный интеллект эффективно считывал контекст кода, технические описания рекомендуется размещать строго до декларации функции, как это принято в большинстве других языков программирования.


```python
def object_to_schema_data(
    obj: Any,
    *,
    schema_field_names: Iterable[str],
    list_related_fields: Iterable[str] = (),
) -> dict[str, Any]:
    """Преобразовать Django-объект в словарь для Pydantic.

    Args:
        obj: Django-модель или словарь.
        schema_field_names: Поля целевой Pydantic-схемы.
        list_related_fields: Имена reverse-связей, которые нужно принудительно
            материализовать как список через `.all()`.

    Returns:
        Словарь данных, подходящий для `model_validate`.
    """
```
