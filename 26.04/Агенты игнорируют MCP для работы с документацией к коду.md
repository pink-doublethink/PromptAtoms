## Ложная задача для ИИ
-----
Автор критикует идею создания MCP-инструментов для передачи ИИ-агентам документации к коду по двум причинам. 

Во-первых, это подменяет реальную навигационную задачу агента (поиск релевантных модулей) на ложную (чтение документации для понимания классов). 

Во-вторых, современные LLM проходили масштабное обучение на базе FIM, но не обучались работе с MCP, поэтому они закономерно игнорируют этот инструмент, предпочитая формировать рабочий контекст привычным для себя образом.
