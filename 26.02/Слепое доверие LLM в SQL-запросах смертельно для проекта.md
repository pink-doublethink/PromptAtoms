## почему слепое доверие LLM в SQL-запросах — это «самоубийство» для проекта
-----
Автор резко критикует рекомендацию использовать Claude для написания SQL-запросов «не думая», называя этот подход опасным для сложных баз данных.

Аргументируется это тем, что LLM воспринимают SQL как «черный ящик» и часто допускают трудноуловимые ошибки в логике (например, при JOIN), которые сложно отладить. 

Подчеркивается, что подобные советы от специалиста с профилем фронтенд-разработчика могут привести к критическим проблемам в продакшене.
