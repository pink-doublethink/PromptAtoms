## почему Waterfall эффективен, а советы по Git и автофиксу требуют осторожности
-----
Автор подвергает критике компетенции Бориса Черного в разработке ядра Claude Code, но подробно разбирает его рекомендации по использованию ИИ-агента.

Из всех советов автор полностью поддерживает только один: необходимость детального проектирования перед генерацией кода (подход Waterfall), так как это эффективнее итеративной отладки. 

Остальные рекомендации — использование параллельных Git-веток, хранение контекста в Markdown и расширение skill-промптов — оцениваются как спорные или недостаточно эффективные по сравнению с альтернативами (графовая память, динамические промпты). 

Подчеркивается, что «автоисправление» багов силами ИИ работает только при наличии мощной инфраструктуры (автотесты, логи), а для улучшения самих промптов автор рекомендует использовать Gemini, а не Claude.
