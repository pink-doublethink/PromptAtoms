## Почему Claude Code не нужен векторный поиск
-----
Отсутствие векторного поиска в Claude Code оправдано: в сложных проектах основной кодирующий ИИ-агент не занимается самостоятельным поиском информации. Вместо этого он получает готовые задания с нужными целевыми файлами от агентов-Архитекторов или Тестировщиков. 

Для сохранения высокого IQ модели Архитектору требуется максимально чистый и короткий контекст (до 8k токенов), который предварительно собирают дешевые субагенты с помощью небольших графов, Doxygen XML или семантического поиска через grep.

Тестировщики же навигируются по коду напрямую из логов.В такой многоагентной системе сложные промежуточные методы поиска становятся вторичными, поскольку нейросети обучены доверять первоисточнику — самому коду.
