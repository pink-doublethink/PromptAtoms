## удобная декомпозиция или поверхностная методика с недостатками?
-----
Автор признает эффективность Spec Driven Development (SDD) для работы с LLM благодаря удобной декомпозиции задач на уровне сниппетов кода. 

Однако методика подвергается критике как «инструмент для новичков»: отмечается недостатком внимания к сложности технологического стека, слабая поддержка контрактного программирования и игнорирование возможностей GPT по анализу логов при тестировании.
