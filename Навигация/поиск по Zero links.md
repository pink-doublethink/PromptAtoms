## Для разового поиска используйте строку поиска
______________
**Она находиться в левом верхнем углу**, где иконка Лупа.

1) Для этого указываем нужное считание заметок. Например поиск:

```
"[[0 Физические компоненты робототехники]]]" OR "[[0 Военные системы]]" OR "[[0 Робототехника]]"
```

Выдаст все заметки где упоминается хотя бы одна из категорий: Робототехника, Физические компоненты робототехники или Военные системы.

2) Если поставить перед словом в ""  знак минус, то:

```
"[[0 Проблемы и вызовы в образовании]]" OR -"[[0 Образовательные подходы]]"
```

Будут показаны все заметки где указана 0 Проблемы и вызовы в образовании и где точно отсутствует категория 0 Образовательные подходы

## Для постоянных подборок
______________

### Через **Dataview-запросы**

Если установлен плагин Dataview, можно искать заметки, которые ссылаются на корневую заметку. Dataview индексирует `file.outlinks` — список исходящих ссылок из файла, и `file.inlinks` — входящие ссылки на файл.

#### Все заметки, связанные с “Контрактное программирование”

```
LISTWHERE contains(file.outlinks, [[0 Контрактное программирование]])
```

#### Все заметки, связанные хотя бы с одной из 4–5 корневых заметок

```
LISTWHERE contains(file.outlinks, [[0 Математические концепты]])   OR contains(file.outlinks, [[0 Эмерджентные явления и новое поведение]])   OR contains(file.outlinks, [[0 Лингвистические и синтаксические концепты]])   OR contains(file.outlinks, [[0 Внутренние процессы и состояния модели]])
```

#### Только заметки, где есть все указанные классификаторы

```
LISTWHERE contains(file.outlinks, [[0 Прогнозы и последствия]])  AND contains(file.outlinks, [[0 Глобальные процессы и тренды]])
```

#### Есть “Образовательные подходы”, но нет “ИИ-модели и системы”

```
LISTWHERE contains(file.outlinks, [[0 Образовательные подходы]])  AND !contains(file.outlinks, [[0 ИИ-модели и системы]])
```

### Через **Bases-представления**
______________
Если вы используете свежий Obsidian с **Bases**, можно сделать таблицу или карточки заметок и фильтровать по ссылкам без Dataview. В Bases есть `file.hasLink(...)`, а также логика `and`, `or`, `not`; в официальной документации прямо показаны фильтры вида `file.hasLink("Textbook")` и отрицание через `not`.

Например, база для заметок, где есть ссылка на “0 Образовательные подходы”, но нет ссылки на “0 ИИ-модели и системы”:

```
filters:  and:    - file.hasLink("0 Образовательные подходы")    - not:      - file.hasLink("0 ИИ-модели и системы")views:  - type: table    name: "Образовательные подходы без ИИ"    order:      - file.name      - file.folder      - file.mtime
```

Для поиска по нескольким корневым заметкам:

```
filters:  or:    - file.hasLink("0 Робототехника")    - file.hasLink("0 Процедурные подходы")    - file.hasLink("0 Образовательные подходы")    - file.hasLink("0 ИИ-модели и системы")views:  - type: table    name: "Связанные с выбранными корнями"    order:      - file.name      - file.folder      - file.mtime
```

В документации Bases также указаны `file.links` и `file.backlinks`, но `file.backlinks` помечен как более тяжёлый по производительности; где возможно, лучше проверять исходящие ссылки через `file.hasLink(...)` / `file.links`.
