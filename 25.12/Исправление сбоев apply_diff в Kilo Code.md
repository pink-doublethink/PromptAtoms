## жесткие правила промптинга для корректных патчей
-----
В моделях Kilo Code наблюдаются критические сбои инструмента apply_diff: допустив ошибку единожды, модель начинает тиражировать неверный формат, требуя перезапуска сессии. Для устранения проблемы предложена инструкция по промптингу. 

Ключевые требования: параметр diff обязан начинаться строго с <<<<<<< SEARCH без лишних символов, а при замене блоков кода необходимо использовать уникальные семантические якоря (START/END), чтобы избежать ошибок позиционирования строк, свойственных LLM
